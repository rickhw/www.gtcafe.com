Day 2: Cargo Hello

用 Cargo 建立專案

cargo 是 rust 的專案樣板工具, 可以透過它建立專案，大概類似於 dotnet 的 project template, java 的 maven ... 等。

$ cargo new hello_cargo
     Created binary (application) `hello_cargo` package

$ ls -ls
total 0
0 drwxr-xr-x  4 rickhwang  staff  128 Feb 22 17:26 hello_cargo

$ cd hello_cargo

$ tree
.
├── Cargo.toml
└── src
    └── main.rs

裡面 檔案說明：

  • Cargo.toml: 存放專案的 metadata, 像是版本, 依賴管理, 編譯等資訊. 後面說明
  • src/main.rs: hello rust 主程式

探索

編譯，並執行：

$ cargo build
   Compiling hello_cargo v0.1.0 (/<username>/repos/rust/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 0.18s

編譯好的 binary 放在 target/debug/ 裡，可以這樣執行：

$ ./target/debug/hello_cargo
Hello, world!

也可以透過 cargo run 執行

$ cargo run
    Finished dev [unoptimized + debuginfo] target(s) in 0.00s
     Running `target/debug/hello_cargo`
Hello, world!

輕鬆愉快, 不需要再額外裝啥工具, 就可以直接跑專案的程式.

Notes

如果修改完 code 之後，直接跑 cargo run 不需要跑 cargo build.

其中 Cargo.lock 並沒有被清掉.

cargo check

cargo 提供快速檢查原始碼的指令，如下：

$ cargo check
    Checking hello_cargo v0.1.0 (/<username>/repos/rust/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 0.02s

如果我把 main.rs 改成這樣：

fn main() {
    println!("Hello, world!"
}

再跑一次：

$ cargo check
    Checking hello_cargo v0.1.0 (/<username>/repos/rust/hello_cargo)
error: mismatched closing delimiter: `}`
 --> src/main.rs:2:13
  |
1 | fn main() {
  |           - closing delimiter possibly meant for this
2 |     println!("Hello, world!"
  |             ^ unclosed delimiter
3 | }
  | ^ mismatched closing delimiter

error: could not compile `hello_cargo` (bin "hello_cargo") due to previous error

官方文件說，通常 cargo check 會比 cargo build 還快, 有時候編譯需要一點時間, 可以先透過 cargo check 快速檢查.

Notes

Cargo 針對不同的作業系統，提供同樣的操作，所以指令都一樣，官方文件不會特別描述。

Cargo.toml

Rust 專案使用 Cargo.toml 為專案設定檔案，內容如下：

[package]
name = "hello_cargo"
version = "0.1.0"
edition = "2021"

# See more keys and their definitions at https://doc.rust-lang.org/cargo/reference/manifest.html

[dependencies]

toml (Tom’s Obvious, Minimal Language) 裡面描述 package 和 dependencies 兩個部分，欄位詳細可以參閱 manifest
