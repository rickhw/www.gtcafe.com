NAS 的用途

May 6, 2024 · 4 min read
Rick Hwang
GTCafe Studio

上次寫了兩篇 NAS 規劃的概念：第一原則 資料分層、第二原則 重要性與存取頻率
接著來寫寫 NAS 採購的重點 - 用途

對我來講，NAS 有兩個主要用途

  1. 資料備份：Storage
  2. 學習用：虛擬機 (VMWare, Container) ---> Computing and Networking

先講講資料備份, 學習用之後再敲

備份習慣的養成​

從開始接觸電腦之後
是剛開始在倚天中文系統學習打字
那時候拿 FFV 故事攻略
當打字練習
也開始「打」日記

過沒多久
開始在 DOS 環境玩編曲
(這是另一個故事)
有一些自己做出來的東西

這時候已經意會到一件事：

如果電腦壞了，我做出來的東西是不是就不見了？

所以資料需要備份
不過那時候我其實連 DOS 的一些基本操作都不會
什麼複製 / 建立目錄 / 搬移 ... 都不會。
但還沒有什麼行動

中毒事件​

果然沒過多久，
同學跟我說，有一個神奇的東西
可以讓電腦發出聲音，
就是只要敲一些指令，
就可以發出音高 ...
我很高興玩了一個下午，
把鋼琴名曲「夢中的婚禮」的第一段輸入
然後電腦內建喇叭發出一些單音聲音
但是音符長度很怪的旋律
很高興，但接下來馬上迎來災難：
就是中毒了，
隔天電腦打開就進入一個我看不懂的畫面。

後來我才知道，
那個讓電腦唱歌的指令搞，
叫做 QBasic

那些重灌的日子​

大概經過一段時間，
經過病毒的洗禮，以及大補帖流行的年代，
開啟重灌人生，從 Dos 開始安裝 -> Win3.1
安裝顯示卡驅動程式，成功在 Win3.1 驅動顯示卡後 (16bit)
在安裝 Win95OSR2 升級版，從 Win3.1 升級到 Win95 ...
這段大概裝上百次之後，才搞懂整個規則。

這個過程，大概也知道要把硬碟分成 C / D 槽
資料隔離概念也摸索出來，
同時也把一些東西複製到磁碟片備份
備份的概念也這樣養成.

這個過程我也開始學 linux
曾經想用 linux 當主力
過程中曾經在 win / linux 之間轉換
搞多重開機
一個不小心 資料又不見了 XD
後來不在同一顆硬碟裝兩個 OS
而是找一台舊電腦裝 linux 跑 LAMP
(記得 P-100 記憶體搞到 96MB, 當時一般電腦頂規 PC 是 P-500 128MB )

固定備份​

後來開始有 CD-R 出現
備份就變成一種習慣了，
光碟的備份是固定頻率的，
每月 / 每兩月 / 每季 ... 等頻率不等.
但就是固定會做的事情.

每次備份的內容
大概就是往前推幾個月所產生的資料
或者有記憶點的資料
所以每張備份，
大概都會有核心資料、新資料 ... 等

直到 2016 年
買了第一台 NAS - QNAP TS-257A

補充：

資料備份背後隱含著
你如何看待資料？
講白話，有一堆資料，你會怎麼整理？分類？管理？
幾個 MB / GB / TB 整理的複雜度
會有所差異

當發生故障的時候
風險的承擔能力？
如何設計一個「可以執行的資料搬移方法」？
資料的差異對你來講重要？
... 等

如果已經有既定的方法
再考慮是否要買 NAS
因為他跟光碟 備份
可以有不一樣的想法

測驗自己對資料整理與分類，
給個任意時間、或者主題
可以很輕鬆的找到
那就是好的整理

也因此
我才有 第一原則 資料分層、第二原則 重要性與存取頻率 這兩篇的原則.
