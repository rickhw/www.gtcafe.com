租屋事件紀錄

June 26, 2026 · 6 min read
Rick Hwang
GTCafe Studio

在外面租房子，總會遇到一些奇怪的事。這些事回想起來，當下都心情有點影響，但事後看都不算太嚴重。

事件一：被檢舉違章建築​

我在南軟工作時期，住在中研院附近的公寓隔間出租套房裡。這出租套房是公寓的四樓兩戶，加上五樓頂加，每戶隔成四間獨立套房，所以共有十六間出租套房。我搬進去住的時候，出租套房剛剛施工完，房間以及相關設施都是新的。

搬進去的第一個禮拜，就收到一張工務局 (?忘了) 的稽查單，詳細內容不太記得了，簡單說就是這層公寓是違章建築，所以市府會派人來稽查。所有房客看到都傻眼，因為大部分的人都剛簽約、搬進來住。剛開始大家還會想聯合起來，但現實結果是：鳥獸散。因為沒人想去搞事，大部分的人摸摸鼻子，就另外找房子去了。

收到稽查單那幾天聯絡不到房東，以為他跑路了，為了避免麻煩，我只好準備再搬家。過了幾天，房東自己出現了，說家裡剛好有事，所以他沒回電話。然後工務局 / 鄰居他去打理好了，跟房客說沒事，可以安心住。但也因為這個事件，有一半的人因此解約、搬走。後來才聽說事情起因是房東在施工過程，沒有跟樓下鄰居打招呼，鄰居不爽去檢舉。

當時我想想，公寓套房地處台北市的邊疆地帶，離南軟很近 (騎車 10 分鐘)，房租算便宜 (記得是 6800)，而且是新蓋好，後來就打消搬走念頭，就住下去了。缺點大概就是潮濕 (靠近汐止) + 冬天超冷。

大概住了兩年，房東把房子賣給一個南部的投資客，所以過程就換了房東，不過房租都沒有漲。

事件二：隔壁租客失蹤​

同樣是發生在中研院出租套房。

有一次下班回去，有一個男生站在公寓一樓門口，好像在等人。一開始我以為他是鑰匙掉了，問了一下，才知道他在等人，我不以為意。隔天下班，又發現他在門口等，就這樣連續幾天，都在門口遇到他。後來稍微問一下，他才說在等他女朋友，已經聯絡不到好幾天了，所以他才在門口看看能不能遇到。繼續問下去才發現，他女友住我隔壁房。。。(我已經全身發毛。。。)

後來我就說直接幫他聯絡房東，叫房東來開門 + 當見證人，同時心裡閃過好幾個後續的劇本。開門的那一瞬間，我們都抖得要死。打開房門之後，裡面沒有人，但有三隻關在籠子裡的波斯貓，已經奄奄一息，快餓死了。大家鬆了一口氣，我也鬆了好大一口氣。。。

後來聯絡到那個女生，結果她自己跑去宜蘭玩一個禮拜，手機沒電了。。。

事件三：住戶電腦中 ARP Spoofing​

租屋套房的網路，就是所有租客共用一個內網。所以只要有人的電腦有狀況，整個出租套房的房客就會雞飛狗跳。

那時候曾遇到其他房客中了 ARP Spoofing 病毒，導致全部租戶的電腦都無法上網。原理是中毒電腦會不斷亂發 ARP 廣播封包，讓區網電腦的 Gateway / DNS IP 都指向中毒的電腦 (實際上是 Mac Address 被改了)，然後連外的封包都往他的電腦送。因為中毒的電腦本身沒有 Gateway / DNS 反查功能，所以整個區域網路都無法上網，只有中毒的人自己可以上網。

我發現之後，搞懂狀況與原理，暫解就是寫個 arp 靜態設定 script，鎖定上游 Router 的 IP / Mac Address，讓電腦每次開機時自動執行，但這只能是暫解。後來找到一個工具叫做 NetCut，運作原理也是利用 ARP Spoofing 干預特定電腦。然後我就用這工具直接把中毒的電腦鎖死。

因為大家都無法上網，所以就開始有人跟房東反應，房東完全不知道怎麼處理。我就主動跟房東說有人中毒了，我知道是哪一個 IP，可以幫忙查。後來我就藉此跟房東要了路由器的帳號密碼，然後也得到房東的授權，去每一戶找，看是誰的電腦中毒。這種查線路的事，跟學生時代在計算機中心打工 (突然想起有這段經驗 XD) 時差不多，經常要去宿舍幫同學查網路問題。

很快的就找到中毒的人，就在我隔壁。他的電腦就是整天掛機刷經驗值，中毒了完全不知道。

整件事情，從發生到發現，到處理完，大該兩個禮拜。

自己的無線網路​

出租套房通常會提供第四台 Cable 和有線網路，但是 WiFi 就要自己想辦法。

還沒有自己弄 Wi-Fi 之前，電腦只能走房東提供的有線網路。那時候剛買了第一隻 Smart Phone (HTC Legend)，為了讓手機可以連 WiFi，就在 PC 上裝一個 USB 無線網卡，然後在 Windows 裝一個軟體 (叫啥忘了) 讓整台 PC 變成路由器 ... 但電腦要一直開著。後來乾脆買一台有 Wi-Fi 的 Router (D-Link XXX)，在房間裡搞一層自己的 LAN (Double NAT)，這樣就解決無線網路的需求，同時也讓自己的網路跟其他人隔離。

那時候工作在做 IoT 的產品，需要搞懂雙層 NAT 怎麼運作，也需要測試 IPCam 怎麼打穿雙層 NAT 的限制，然後才能做 Live Stream 傳輸。也算是個生活跟工作直接有關係的經驗。也因為在租屋處本來就有類似環境，所以也讓工作時更快進入狀況。

常見狀況​

台北市一堆公寓改建成出租套房，大多都是投資客在搞的。公寓改建的套房我遇過的狀況：

  1. 進出不關大門、房門
  2. 公共區域髒亂
  3. 有人會半夜開 Party、聽到各種奇怪的聲音
  4. 沒有網路、網路不穩、區網有人中毒、沒有 WiFi
  5. 水壓不穩、加壓馬達很吵
  6. 曬衣空間不夠

可能還有其他的，之後想到再紀錄。
