曲式、歷史背景、框架模式

這幾個背後的本質是類似的。

我很喜歡看金庸武俠小說，其中幾部長篇把歷史真實故事跟虛構的人事物巧妙的結合，是最讓人回為無窮的！像是神雕俠侶、射鵰英雄、鹿鼎記 … 等。

以創作小說來看，歷史已經是一個固定的框架了，創作者能夠發揮的是框架以外的部分或者是一些模糊地帶。這些部分通常是一些野史、傳說、非正式的紀錄。創作者透過自己的聯想、構思、虎爛，把真實的歷史，加上模糊地帶的野史，最後至入完全虛構的人事物，構成一篇讓人讀起來回為無窮的小說！

我想從金庸的封筆之作 - 鹿鼎記，裡就可以看出很多這樣的例子。有時候看完鹿鼎記，我都會懷疑，韋小寶這傢伙或許是存在的。當然一方面這表示我入戲太深，另一方面就是查爺爺整合真實與虛擬的功力實在是一級棒！(或者說虎爛的功夫一級棒XD)

音樂這種藝術也是有類似的形式。常見的大概就是曲式 (Form)、和弦進行、節奏 Style、Loop。曲式最常見的爵士標準曲 (Jazz Standard)、藍調 16、12、8 小節，流行歌曲常用的降冪和弦進行，卡農曲式；各式各樣的音樂風格都可以說是一種固定形式！

當然啦，歷史有的時候也會被推翻，所以偶爾會有改寫歷史的狀況。音樂當然也是，只是不能說是推翻，因該是一種創新吧！算是新的曲式、新的風格～

著名的 Ⅱ-Ⅴ-I 進行就有千千萬萬首歌曲 (Realbook 裡少說數百首)，最常見的例如:

  1. Autumn Leaves
  2. All the things you are
  3. 黃品源 - 你怎麼捨得我難過
  4. 西洋經典老歌 - Hello
  5. Gary Moore - Still Got the Blues …

但是每一首的感覺都不太一樣。

虎爛到最後我還是要回到我的本行，因為不管是歷史的框架、還是音樂的曲式，都沒有軟體的框架模式 (Framework) 多 … 不管是什麼樣的技術都有所謂的 Spec。都會有所謂的 Framework。Spec還好，大部分是有聯合廠商的非官方機構定義，像是 Java 的 JCP。可是 Framework 就慘了，每一家廠商都自己的。說得出來的 IT 大廠都會有自己的 Framework。光列出名稱就下班了。所以 … 還是音樂比較好玩 … (上班發牢騷XD)
