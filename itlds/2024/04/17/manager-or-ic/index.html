管理職 or 獨立貢獻者

從 #管理職 (Manager) 換成 #獨立貢獻者 (IC, #IndividualContributor) 之後的差異

最大的改變: #會議變少 #時間變多 (Calendar 變超空)

會議代表情報變少
代表有了資訊落差
「我知道你不知道的」這就是落差
情報就會創造落差
(情蒐就是一種能力的體現 / 或者條件)

也就是重要的事情
IC 通常不會一起參與決策與討論
而是變成事後被告知
說得好聽點叫做諮詢 / Consult
其實是人家已經討論一輪的東西
做好決策了
你只是去填個洞而已

Manager 會知道很多有的沒的
像是重要的方向、決策、資源狀況 ..等 #情報
特別是決策過程的 #決策邏輯 ..

IC 本質上就是專注在 #執行面
用自身的技能滿足目標

從 Manager 轉換成 IC 之後
會議變少, 時間變多
已經很習慣 Context Switch
突然不知道要 Switch 啥
但是腦袋還是在 Empty Switch
停不下來 ..

有好有壞 ..
喜歡刷存在感 / 幹大事
就去幹 Manager
喜歡踏實做事
就走 IC ..

但因為幹過 Manager
很多事情其實是可以反推出來的
IC 的身份可以給自己的主管建議
而且更可以站在 Manager 的經驗建議
當然 也可以選擇當作不知道
當個旁觀者

其實這是我四年前的狀況 XD ...
卸下 Manager as Architect 的時候
有半年的空窗 ...
不知道自己在做啥
但腦袋也瘋狂的在 Context Switch ..

雖然後來
還是變成 Manager + IC .
IcManager?

台灣 (或者華人圈) 普遍的觀念
年紀越大 越要往管理職
如果到了一定年紀 (40+)
名片拿出來 是「OOXX 工程師」
好像就是矮人一截
名片寫「OOXX 經理」就高級多了 ..

我覺得是沒啥意義 ..
對我來講最好的 Title 就是: Rick ....
